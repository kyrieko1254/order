# 커피 주문 앱 PRD (Product Requirements Document)

## 1. 개요
- 본 앱은 사용자가 커피 메뉴를 주문하고, 관리자가 주문 및 재고를 관리할 수 있는 간단한 커피 주문 시스템이다.
- 최소한의 기능만 구현하여 풀스택 앱 개발을 학습하는 데 목적이 있다.

## 2. 주요 사용자
- **일반 사용자**: 커피 메뉴를 선택하고 주문할 수 있음
- **관리자**: 주문 현황 및 재고를 관리할 수 있음

## 3. 주요 기능

### 3.1 사용자(고객) 화면
- 메뉴 리스트 조회
- 메뉴 옵션(사이즈, 추가 샷 등) 선택
- 수량 선택
- '담기' 버튼 클릭 시 장바구니에 메뉴 추가
- 장바구니에서 주문 내역 확인 및 주문 확정

### 3.2 관리자 화면
- 전체 주문 목록 확인 (주문 상태별 필터링)
- 각 주문의 상세 정보(메뉴, 옵션, 수량, 주문 시간, 상태 등) 확인
- 주문 상태 변경 (예: 접수, 준비 중, 완료 등)
- 각 메뉴별 재고량 확인 및 수정

## 4. 화면 흐름

### 4.1 사용자 플로우
1. 앱 접속 → 메뉴 리스트 확인
2. 메뉴 및 옵션 선택 → '담기' 클릭 → 장바구니에 추가
3. 장바구니에서 주문 내역 확인 → 주문 확정

### 4.2 관리자 플로우
1. 관리자 로그인(간단한 인증 또는 무인증)
2. 주문 목록 및 상태 확인
3. 주문 상태 변경(예: 접수 → 준비 중 → 완료)
4. 메뉴별 재고량 확인 및 수정

## 기술 스택
- 프런트엔드: React, Javascript
- 백엔드: Node.js(Express) 등
- DB: PostgreSQL
